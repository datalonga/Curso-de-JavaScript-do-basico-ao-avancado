Tópico 1. O que é JavaScript?

menu_book
Definição Técnica
JavaScript é uma linguagem de programação de alto nível, interpretada e de múltiplos paradigmas. Originalmente criada para rodar em navegadores web, ela permite adicionar comportamento dinâmico a páginas estáticas. Enquanto o HTML define a estrutura do conteúdo e o CSS define a apresentação visual, o JavaScript é responsável pela lógica e interatividade. Sem ele, os sites seriam documentos estáticos, semelhantes a PDFs na tela.
lightbulb
Analogia Técnica
Imagine um automóvel: HTML é a estrutura metálica e a carroceria (o que existe fisicamente). CSS é a pintura, os acabamentos e o design (como aparece visualmente). JavaScript é o motor, a transmissão e os sistemas eletrônicos (o que faz o carro se mover, responder ao volante e executar funções).

Para que serve?

JavaScript é utilizado em praticamente todas as camadas do desenvolvimento moderno:

  • Front-end: Interatividade em sites, validação de formulários, animações e aplicações web completas (SPAs).
  • Back-end: Servidores e APIs usando ambientes como Node.js.
  • Mobile: Aplicativos nativos e híbridos (React Native, Ionic).
  • Automação & Scripts: Tarefas repetitivas, extensões de navegador e ferramentas de desenvolvimento.

Nota Histórica

Criado por Brendan Eich em 1995 para o navegador Netscape, o JavaScript foi desenvolvido em apenas 10 dias. Apesar do nome, não possui relação direta com a linguagem Java. O nome foi uma decisão de marketing da época.

Primeiro Exemplo Prático

O comando console.log() é a forma padrão de exibir informações no console do navegador ou do terminal. Ele é essencial para depuração (debugging) e testes iniciais.

javascript — Primeiro Script
// Comentários são ignorados pelo interpretador.
// Servem para documentar o código.

console.log("JavaScript carregado com sucesso.");
console.log("Executando primeira instrução.");
console.log(42 + 8);
warning
Atenção
JavaScript é case-sensitive. console.log funciona, mas Console.Log ou CONSOLE.LOG gerarão um erro de referência (ReferenceError). A sintaxe deve ser escrita exatamente como definida pela linguagem.

Como Executar

  • Console do Navegador: Pressione F12 ou Ctrl+Shift+I, acesse a aba "Console" e digite o código diretamente.
  • Playgrounds Online: Plataformas como CodePen, JSFiddle ou Replit permitem escrever, executar e salvar código sem configuração local.
  • Node.js: Ambiente de execução que roda JavaScript fora do navegador, ideal para back-end e scripts.

Tópico 2. Variáveis

menu_book
Definição Técnica
Variáveis são identificadores nomeados que armazenam referências a valores na memória do computador. Em vez de trabalhar diretamente com endereços de memória complexos, o programador utiliza nomes simbólicos para ler e modificar dados durante a execução do código. O JavaScript é uma linguagem dinamicamente tipada, o que significa que o tipo do valor é inferido automaticamente em tempo de execução.
lightbulb
Analogia Técnica
Imagine um sistema de armazenamento organizado: cada caixa possui uma etiqueta (nome da variável) e um conteúdo (valor). Quando você precisa do conteúdo, não procura em todas as caixas; basta ler a etiqueta.

Declaração: let, const e var

  • let: Declarada para valores que serão modificados posteriormente. Possui escopo de bloco.
  • const: Declarada para valores que não devem ser reatribuídos. Possui escopo de bloco. É a escolha padrão.
  • var: Forma legada. Possui escopo de função e sofre de hoisting (elevação), o que frequentemente gera bugs difíceis de rastrear. Evite em código moderno.
check_circle
Boa Prática
Comece declarando tudo com const. Se a lógica do programa exigir que o valor mude, altere para let.

Regras de Nomenclatura

Devem começar com letra, _ (underscore) ou $ (cifrão). Não podem conter espaços ou começar com números. São case-sensitive. Utilize camelCase para variáveis comuns.

javascript — Nomenclatura
// Válido
let nomeCompleto = "Ana Silva";
const idade = 10;
let _contador = 0;

// Inválido (gera erro de sintaxe)
// let 2idade = 10;       // Não pode começar com número
// let nome completo = "X"; // Não pode conter espaços
// let class = "A";       // Palavra reservada

Atribuição e Reatribuição

O operador = em JavaScript não representa igualdade matemática, mas sim atribuição.

javascript — Atribuição
const nivelBateria = 85;
let usuarioLogado = true;
let mensagemPadrao = "Sistema inicializado.";

// Reatribuição válida (let)
mensagemPadrao = "Atualização aplicada.";
usuarioLogado = false;

// Reatribuição inválida (const) - gera TypeError
// nivelBateria = 50;
warning
Atenção: let vs const em Objetos
const impede a reatribuição da variável, mas não congela o conteúdo de objetos ou arrays. Você pode modificar propriedades internas de um objeto declarado com const.

Tópico 3. Tipos de Dados

menu_book
Definição Técnica
Todo valor armazenado pertence a uma categoria chamada tipo de dado. JavaScript classifica os dados em primitivos (imutáveis) e objetos (estruturas complexas). Os 5 tipos primitivos essenciais são: string, number, boolean, undefined e null.
lightbulb
Analogia Técnica
Imagine um painel de controle:
string = Etiquetas textuais coladas nos botões.
number = Mostradores analógicos (velocidade, temperatura).
boolean = Interruptores de duas posições (Ligado/Desligado).

Tipos de Valor: string, number e boolean

javascript — Declaração dos tipos
const nomeJogo = "Space Defender";   // string
const pontuacao = 1250.75;           // number
const estaOnline = true;             // boolean
const tentativasRestantes = 3;       // number

Ausência de Valor: undefined e null

  • undefined: Atribuído automaticamente quando uma variável não recebe conteúdo inicial.
  • null: Atribuído intencionalmente pelo programador para indicar ausência deliberada.
javascript — undefined vs null
let emailConfirmado;
console.log(emailConfirmado); // undefined (padrão)

emailConfirmado = null;
console.log(emailConfirmado); // null (definido pelo dev)

Verificação de Tipos com typeof

javascript — Operador typeof
console.log(typeof "JavaScript");  // "string"
console.log(typeof 42);           // "number"
console.log(typeof true);          // "boolean"
console.log(typeof undefined);     // "undefined"
console.log(typeof null);          // "object" (herança histórica da linguagem)
warning
Atenção: Coerção de Tipos
JavaScript tenta converter tipos automaticamente em operações mistas. Exemplo: "5" + 3 resulta em "53" (concatenação de strings), enquanto "5" - 3 resulta em 2 (conversão para número).

Tópico 4. Operadores

menu_book
Definição Técnica
Operadores são símbolos ou palavras-chave que executam operações sobre operandos. Cada categoria possui prioridade de execução (precedência), determinando a ordem de resolução em expressões.

Operadores Aritméticos

Realizam operações matemáticas básicas sobre valores numéricos: adição (+), subtração (-), multiplicação (*), divisão (/) e potência (**). O operador % (módulo) retorna o resto da divisão inteira, sendo amplamente utilizado para verificar paridade.

javascript — Aritméticos
const base = 15;
const divisor = 4;

console.log(base + 5);       // 20 (adição)
console.log(base - 5);       // 10 (subtração)
console.log(base * 2);       // 30 (multiplicação)
console.log(base / divisor); // 3.75 (divisão)
console.log(base % divisor); // 3 (resto da divisão)
console.log(2 ** 8);         // 256 (potência)

Operadores de Atribuição

Além do operador básico =, o JavaScript oferece atalhos que combinam atribuição com outra operação matemática.

javascript — Atribuição Composta
let pontos = 100;

pontos += 25;  // Equivalente a: pontos = pontos + 25;
console.log(pontos); // 125

pontos -= 10;  // Equivalente a: pontos = pontos - 10;
console.log(pontos); // 115

pontos *= 2;   // Equivalente a: pontos = pontos * 2;
console.log(pontos); // 230

Operadores de Comparação

  • === (Estrita): Verifica valor e tipo. Recomendado em 99% dos casos.
  • == (Solta): Verifica apenas valor, aplicando coerção de tipos. Pode gerar bugs.
  • !== / !=: Versões negadas, respectivamente estrita e solta.
javascript — Comparação
console.log(5 === 5);       // true (mesmo tipo e valor)
console.log(5 === "5");     // false (tipos diferentes)
console.log(5 == "5");      // true (coerção aplicada)
console.log(10 > 7);        // true
console.log(3 <= 3);        // true
check_circle
Padrão de Mercado
Utilize sempre === e !==. A igualdade solta realiza conversões que mascaram erros de lógica.

Operadores Lógicos

  • && (AND): Retorna true apenas se todas as condições forem verdadeiras.
  • || (OR): Retorna true se pelo menos uma condição for verdadeira.
  • ! (NOT): Inverte o valor lógico da expressão.
javascript — Lógicos
const temCarteira = true;
const temIdade = 18 >= 18; // true

// Ambas devem ser verdadeiras
const podeDirigir = temCarteira && temIdade;
console.log(podeDirigir); // true

// Inversão lógica
console.log(!podeDirigir); // false

Tópico 5. console.log e Depuração Básica

lightbulb
Analogia Técnica
O console é a caixa-preta do seu código. Ela não controla o sistema, mas registra dados críticos em tempo real.

Formatação com Template Literals

A forma moderna de construir saídas dinâmicas utiliza template literals (delimitados por crases ` `). A sintaxe ${expressão} interpola variáveis diretamente.

javascript — Interpolação de strings
const nivel = 7;
const experiencia = "intermediário";

// Método antigo (concatenação)
console.log("Nível: " + nivel + " | Perfil: " + experiencia);

// Método moderno (template literal)
console.log(`Nível: ${nivel} | Perfil: ${experiencia}`);
console.log(`Progresso: ${nivel * 10}%`);

Métodos Auxiliares do Console

  • console.warn(): Registra avisos em amarelo.
  • console.error(): Registra falhas críticas em vermelho.
  • console.table(): Formata arrays ou objetos em uma tabela legível.
  • console.time() / console.timeEnd(): Medem tempo de execução.
javascript — Métodos do console
// Registro estruturado
console.warn("Bateria abaixo de 15%. Recarregue.");

// Tabela para inspeção visual
const jogadores = [
  { nome: "Ana", pontos: 1250 },
  { nome: "Leo", pontos: 980 }
];
console.table(jogadores);

Tópico 6. Condicionais (if / else)

Condicionais são estruturas de controle de fluxo que permitem executar blocos distintos com base na avaliação booleana. O fluxo avança apenas quando a condição é satisfeita.

javascript — Estrutura condicional
const nivelAcesso = "editor";

if (nivelAcesso === "admin") {
  console.log("Acesso total concedido.");
} else if (nivelAcesso === "editor") {
  console.log("Edição habilitada. Permissão limitada.");
} else {
  console.log("Acesso negado. Contate o administrador.");
}

Valores Truthy e Falsy

  • Falsy: false, 0, "", null, undefined e NaN.
  • Truthy: Tudo fora da lista (ex: strings não vazias, arrays vazios [], números diferentes de zero).
javascript — Coerção em condições
if ("texto") console.log("truthy");      // Executa
if (0) console.log("falsy");            // Ignora
if ([]) console.log("truthy");          // Executa (array vazio é truthy!)
check_circle
Boa Prática
Em vez de verificar if (usuario && usuario.ativo === true), extraia a validação: const podeAcessar = usuario && usuario.ativo; e depois use if(podeAcessar).

Tópico 7. Loops (for / while)

Loops automatizam a repetição até que uma condição de término seja atingida. O for é para repetições conhecidas, enquanto o while é para repetições que dependem de um estado.

O Loop for

javascript — Loop for
// Inicializa em 0; executa enquanto i < 5; incrementa i em 1
for (let i = 0; i < 5; i++) {
  console.log(`Iteração número: ${i}`);
}

O Loop while

javascript — Loop while
let energia = 100;
let tentativa = 1;

while (energia > 20) {
  console.log(`Tentativa ${tentativa}: energia em ${energia}`);
  energia -= 15;
  tentativa++;
}

Controle de Fluxo: break e continue

  • break: Interrompe totalmente o loop.
  • continue: Pula o restante da iteração e vai para a próxima.

Tópico 8. Condicionais Múltiplas (switch)

A instrução switch avalia uma expressão e compara o resultado (com ===) com múltiplas cláusulas case. O break é fundamental para evitar a execução encadeada (fall-through).

javascript — Estrutura switch
const statusPedido = "enviado";

switch (statusPedido) {
  case "pendente":
    console.log("Aguardando pagamento.");
    break;
  case "enviado":
    console.log("Em trânsito.");
    break;
  default:
    console.log("Status desconhecido.");
}
warning
Atenção Crítica: Fall-through
Sem o break, o interpretador continua executando sequencialmente os blocos seguintes. Sempre inclua break ao final de cada bloco, a não ser que o agrupamento seja intencional.

Tópico 9. Introdução às Funções

Funções encapsulam lógica repetitiva em uma unidade reutilizável, fundamentando o princípio DRY (Don't Repeat Yourself).

javascript — Declaração vs Invocação
// 1. Declaração (armazena na memória)
function inicializarSistema() {
  console.log("Carregando módulos...");
  console.log("Sistema pronto.");
}

// 2. Invocação (executa o código)
inicializarSistema();

Referência vs Execução

minhaFuncao passa a referência da função (sem executar), enquanto minhaFuncao() diz ao interpretador para executá-la.

Tópico 10. Parâmetros e Retorno

A instrução return devolve um valor para quem invocou a função e encerra a execução da mesma imediatamente.

javascript — Parâmetros e Argumentos
function calcularPotencia(base, expoente) {
  let resultado = 1;
  for (let i = 0; i < expoente; i++) {
    resultado *= base;
  }
  return resultado;
}

const valor = calcularPotencia(2, 10);
console.log(valor); // 1024

Padrão: Guard Clauses (Retorno Antecipado)

Valide condições de falha primeiro e use return para sair cedo. Reduz a indentação.

javascript — Guard Clause & Valores Padrão
function gerarPagina(titulo, idioma = "pt-BR") {
  if (!titulo) return "Erro: Título obrigatório";
  return `Página "${titulo}" em ${idioma}.`;
}

Tópico 11. Arrow Functions (=>)

Uma sintaxe mais compacta e moderna. Elimina a necessidade da palavra-chave function, podendo até suprimir as chaves e o return para casos de uma única linha (retorno implícito).

javascript — Refatoração para arrow function
// 1. Função tradicional
function dobrarValor(n) { return n * 2; }

// 2. Arrow function com bloco explícito
const dobrarValor = (n) => { return n * 2; };

// 3. Arrow function com retorno implícito (one-liner)
const dobrarValor = n => n * 2;
warning
Diferença Crítica: Escopo do This
Diferente das funções tradicionais, Arrow functions não possuem o próprio this. Elas herdam o escopo de onde foram declaradas. Não as use em métodos de objeto que precisam do this.

Tópico 12. Arrays (Vetores)

Arrays são listas ordenadas dinâmicas indexadas por números (começando em zero).

javascript — Declaração e acesso
const protocolos = ["HTTP", "FTP", "SSH", "DNS"];

console.log(protocolos[0]); // "HTTP" (primeiro elemento)
console.log(protocolos[3]); // "DNS" (último elemento)
console.log(protocolos[4]); // undefined (índice inexistente)

Propriedade length

Retorna o tamanho. Lembre-se do "off-by-one error": o último índice de um array sempre será length - 1.

Tópico 13. Métodos de Array

Métodos evitam loops manuais e deixam código declarativo. Dividem-se em mutadores e não-mutadores.

Mutadores (alteram original)

push(), pop(), shift(), splice(), sort().

Não-Mutadores (retornam cópia/valor)

slice(), join(), includes().

Iteração Funcional

javascript — map, filter, find
const valores = [10, 25, 8, 42];

// map: transforma cada valor
const dobrados = valores.map(n => n * 2); // [20, 50, 16, 84]

// filter: mantém apenas valores >= 20
const altos = valores.filter(n => n >= 20); // [25, 42]

// find: retorna o primeiro elemento par
const primeiroPar = valores.find(n => n % 2 === 0); // 10

Tópico 14. Objetos

Estruturas compostas por pares de chave-valor. Ideais para modelar entidades e agrupamentos lógicos.

javascript — Criação e acesso
const configSistema = {
  nome: "Dashboard",
  versao: "2.4.1",
  maxUsuarios: 500,
  ativo: true
};

// Notação de ponto (recomendada)
console.log(configSistema.nome); // "Dashboard"

// Notação de colchetes (para chaves dinâmicas)
const chaveDinamica = "versao";
console.log(configSistema[chaveDinamica]); // "2.4.1"
warning
Referências em Memória
Atribuir um objeto a outra variável apenas copia o apontador. Alterar a cópia também afeta o original (shallow copy). Use o operador Spread { ...obj } para copiar separadamente.

Tópico 15. Métodos de Objeto

Ferramentas para inspecionar, iterar e manipular chaves e valores estruturais dinamicamente.

javascript — Inspeção de dados
const config = {
  tema: "escuro",
  notificacoes: true,
  idioma: "pt-BR"
};

console.log(Object.keys(config));    // ["tema", "notificacoes", "idioma"]
console.log(Object.values(config));  // ["escuro", true, "pt-BR"]
console.log(Object.entries(config)); // [["tema", "escuro"], ["notificacoes", true], ...]

Validação e Cópia

Object.hasOwn(obj, "chave") verifica a existência de forma segura.
O Spread Operator { ...obj1, ...obj2 } é a forma moderna e legível para mesclar objetos.

Tópico 16. O DOM

menu_book
Definição Técnica
O DOM (Document Object Model) converte cada elemento HTML em um nó (node) em memória, criando uma árvore viva. É a ponte entre a estrutura estática (HTML) e a interatividade programável (JavaScript).

Modificar o DOM via JS não muda o seu arquivo HTML original no servidor, ele atualiza apenas a instância em RAM mostrada no navegador (o layout da tela).

javascript — Inspeção e Navegação
console.log(document.title);       // Título atual
console.log(document.body);        // O body inteiro

const app = document.getElementById("app");
console.log(app.children.length);  // Qtos filhos elementos ele tem?
console.log(app.parentNode);       // Quem é o pai do app?

Tópico 20. Promises

Promessas representam operações assíncronas no JS (requisições, timeouts, arquivos). Elas vivem em 3 estados: Pendente, Cumprida (Resolved) e Rejeitada (Rejected).

javascript — Tratamento de estados
buscaDados
  .then((resultado) => {
    // Executado se a Promise for resolve
    console.log("Sucesso:", resultado);
  })
  .catch((erro) => {
    // Executado se a Promise for reject
    console.error("Falha:", erro);
  })
  .finally(() => {
    // Roda em ambos os casos (limpeza e spin loaders)
    console.log("Fim da operação");
  });

Tópico 21. Async/Await

Uma roupagem (syntactic sugar) mais moderna sobre o topo de Promises, para evitar o uso excessivo de encadeamentos .then() tornando o código lido como se fosse linear, síncrono.

javascript — Async / Await com try-catch
async function processarPedido(id) {
  try {
    const pedido = await fetch(`/api/pedidos/${id}`); // pausa aqui
    if (!pedido.ok) throw new Error("Pedido não encontrado");
    
    console.log("Sucesso");
  } catch (erro) {
    console.error("Falha ao processar:", erro.message);
  }
}
check_circle
Boa prática
Abrace blocos try/catch para funções assíncronas. Uma reject escapada sem catch crasha workers e lança avisos perigosos.

const input = document.querySelector("#novaTarefa");
const botao = document.querySelector("#btnAdicionar");
const lista = document.querySelector("#listaTarefas");

// Escuta de delegação de clicks dentro da UL
lista.addEventListener("click", (e) => {
  // Clique no texto
  if (e.target.tagName === "LI") {
    e.target.classList.toggle("concluido");
  }
  
  // Clique no botão X
  if (e.target.classList.contains("btn-remover")) {
    e.target.parentElement.remove();
  }
});

// Criador universal de nós
function adicionarItem(texto) {
  const li = document.createElement("li");
  li.textContent = texto; // Uso do content previne injeção XSS
  
  const btn = document.createElement("span");
  btn.textContent = "×";
  btn.classList.add("btn-remover");
  
  li.appendChild(btn);
  lista.appendChild(li);
}

// Vinculo externo para ativação
botao.addEventListener("click", () => {
  if (input.value.trim() !== "") {
    adicionarItem(input.value);
    input.value = ""; // Limpa Input
  }
});
